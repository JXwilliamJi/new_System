# 移动端响应式适配指南

## 概述

本文档描述了法务合规管理系统的移动端响应式适配方案。该方案旨在提供优秀的移动端用户体验，确保系统在各种移动设备上都能正常显示和使用。

## 适配范围

### 支持的设备
- **手机**: iPhone SE及以上、Android手机（屏幕宽度 ≤ 768px）
- **平板**: iPad、Android平板（屏幕宽度 769px - 1200px）
- **桌面**: 所有现代浏览器（屏幕宽度 > 1200px）

### 支持的浏览器
- iOS Safari 12+
- Chrome 60+
- Firefox 60+
- Samsung Internet 8+
- 微信内置浏览器

## 主要功能

### 1. 移动端导航菜单

**功能特点**:
- 自动添加汉堡菜单按钮（左上角）
- 点击按钮打开/关闭侧边栏
- 点击遮罩层关闭侧边栏
- 向左滑动关闭侧边栏
- ESC键关闭侧边栏
- 窗口大小变化时自动关闭

**使用方法**:
- 在移动端（屏幕宽度 ≤ 768px），左上角会自动显示菜单按钮
- 点击按钮打开侧边栏
- 点击遮罩层或向左滑动关闭侧边栏

### 2. 响应式布局

**断点设置**:
- **大屏幕**: > 1200px
- **中等屏幕**: 769px - 1200px
- **小屏幕**: ≤ 768px

**布局变化**:
- **KPI卡片**: 5列 → 2列 → 1列
- **图表区域**: 2列 → 1列
- **底部网格**: 2列 → 1列
- **数据汇总**: 2列 → 1列
- **工具栏**: 水平 → 垂直

### 3. 表格优化

**移动端优化**:
- 表格容器支持水平滚动
- 最小宽度600px，确保内容可读
- 操作按钮自动换行
- 状态标签字体缩小

**使用方法**:
- 在移动端，表格可以左右滑动查看完整内容
- 操作按钮会自动调整大小和位置

### 4. 表单优化

**移动端优化**:
- 输入框字体大小16px（防止iOS自动缩放）
- 表单按钮全宽显示
- 搜索框和下拉框全宽显示
- 触摸区域最小44px

**使用方法**:
- 表单元素会自动适应移动端屏幕
- 输入时不会自动缩放页面

### 5. 模态框优化

**移动端优化**:
- 宽度95%，最大宽度95%
- 内边距20px
- 最大高度85vh
- 按钮全宽显示
- 关闭按钮定位到右上角

**使用方法**:
- 模态框会自动适应移动端屏幕
- 内容可滚动查看

### 6. 触摸优化

**优化内容**:
- 所有可点击元素最小44px × 44px
- 按钮点击反馈效果
- 卡片点击缩放效果
- 表格行点击高亮效果

**使用方法**:
- 所有按钮和链接都适合触摸操作
- 点击时会有视觉反馈

### 7. 安全区域适配

**适配内容**:
- iPhone X及以上刘海屏
- 底部安全区域
- 顶部安全区域

**使用方法**:

- 底部内容不会被Home指示器遮挡

### 8. 性能优化

**优化措施**:

**效果**:
- 提高移动端滚动流畅度

## 使用说明

### 开发者

#### 1. 添加移动端支持

如果需要为新页面添加移动端支持，需要：

1. 在HTML头部添加以下meta标签：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="theme-color" content="#1a3c6e">
```

2. 引入移动端响应式CSS：
```html
<link rel="stylesheet" href="/css/mobile-responsive.css">
```

3. 在页面底部引入移动端菜单脚本：
```html
<script src="/js/mobile-menu.js"></script>
```

#### 2. 使用移动端工具类

移动端提供了一些实用的工具类：

**显示/隐藏类**:
- `.mobile-hidden`: 在移动端隐藏
- `.mobile-only`: 仅在移动端显示

**间距类**:
- `.mobile-mt-0` 到 `.mobile-mt-4`: 移动端上边距
- `.mobile-mb-0` 到 `.mobile-mb-4`: 移动端下边距
- `.mobile-p-0` 到 `.mobile-p-4`: 移动端内边距

**文本类**:
- `.mobile-text-truncate`: 文本截断

#### 3. 自定义移动端样式

如果需要为特定元素添加移动端样式，可以使用媒体查询：

```css
@media (max-width: 768px) {
    .your-element {
        /* 移动端样式 */
    }
}
```

## 常见问题

### Q1: 为什么输入时页面会自动缩放？

**A**: 这是因为viewport meta标签没有设置`maximum-scale=1.0`和`user-scalable=no`。我们已经添加了这些设置，现在输入时页面不会自动缩放。

### Q2: 为什么侧边栏在移动端显示不正常？

**A**: 侧边栏在移动端会自动隐藏，需要点击左上角的菜单按钮才能打开。如果仍然显示不正常，请检查是否正确引入了`mobile-responsive.css`和`mobile-menu.js`。

### Q3: 为什么表格在移动端无法查看完整内容？

**A**: 表格在移动端支持水平滚动，可以左右滑动查看完整内容。如果仍然无法查看，请检查表格是否设置了最小宽度。

### Q4: 为什么模态框在移动端显示不正常？

**A**: 模态框在移动端会自动调整大小和位置。如果仍然显示不正常，请检查是否正确引入了`mobile-responsive.css`。

### Q5: 如何为新页面添加移动端支持？

**A**: 请参考"开发者"部分的"添加移动端支持"说明。
