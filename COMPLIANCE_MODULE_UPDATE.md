# 合规管理模块更新说明

## 更新内容

本次更新实现了以下三个主要功能：

### 1. 类型字段可配置化

**功能描述：**
- "类型"字段的可选选项现在可以在后台管理中自行编辑和设置
- 默认包含以下类型：合同审查、项目采购、制度审查、合法性审查、法律服务

**实现细节：**
- 新增 `compliance_types` 表，存储合规类型配置
- 在后台管理页面添加"合规类型"配置模块
- 支持添加、编辑、删除合规类型
- 支持设置排序和启用/禁用状态

**使用方法：**
1. 登录后台管理页面
2. 点击左侧菜单"合规类型"
3. 可以添加新类型、编辑现有类型或删除类型
4. 编辑类型时可以选择是否同步更新已归档的项

### 2. 列表加入提交部门和提交人

**功能描述：**
- 合规检查列表新增"提交部门"和"提交人"两个字段
- 提交部门可在后台管理中自行编辑
- 默认包含以下部门：办公室、科技一部、科技二部、业务一部、业务二部

**实现细节：**
- 新增 `departments` 表，存储部门配置
- 在 `compliance_checks` 表中添加 `submit_department` 和 `submit_user` 字段
- 在后台管理页面添加"部门配置"模块
- 合规检查表单中添加部门和用户选择框

**使用方法：**
1. 在后台管理页面点击"部门配置"
2. 可以添加新部门、编辑现有部门或删除部门
3. 编辑部门时可以选择是否同步更新已归档的项
4. 在合规检查表单中选择提交部门和提交人

### 3. 后台修改名称时询问是否同步更新已归档项

**功能描述：**
- 当在后台修改类型或部门名称时，系统会询问是否将已经归档的项名称一同修改
- 如果选择"是"，所有已归档的项都会更新为新名称
- 如果选择"否"，只有新保存的项使用新名称，已归档项保持原名称

**实现细节：**
- 编辑类型/部门时显示"同时更新已归档的合规检查项"复选框
- 后端根据 `update_archived` 参数决定是否更新已归档数据
- 更新成功后会显示相应的提示信息

**使用方法：**
1. 在后台管理页面编辑类型或部门
2. 勾选"同时更新已归档的合规检查项"复选框
3. 保存后，系统会提示是否已同步更新

## 数据库变更

### 新增表

1. **compliance_types** - 合规类型配置表
   - id: 主键
   - name: 类型名称
   - sort_order: 排序
   - is_active: 是否启用
   - created_at: 创建时间
   - updated_at: 更新时间

2. **departments** - 部门配置表
   - id: 主键
   - name: 部门名称
   - sort_order: 排序
   - is_active: 是否启用
   - created_at: 创建时间
   - updated_at: 更新时间

### 修改表

**compliance_checks** - 合规检查表
- 新增 `submit_department` 字段：提交部门
- 新增 `submit_user` 字段：提交人（关联用户表）

## 文件变更

### 新增文件
- `database/migrate_compliance.js` - 数据库迁移脚本

### 修改文件
- `routes/admin.js` - 添加类型和部门管理的API接口
- `public/pages/admin.html` - 添加类型和部门管理的UI界面
- `public/pages/compliance.html` - 更新合规检查页面，添加新字段
- `routes/compliance.js` - 更新合规检查的CRUD操作，支持新字段

   - 添加新部门
   - 编辑部门名称并选择同步/不同步已归档项

   - 修改类型名称并选择同步，检查已归档项是否更新
   - 修改部门名称并选择不同步，检查已归档项是否保持原值

## 注意事项

1. 数据库迁移脚本会自动为现有数据设置默认提交部门（办公室）和提交人（创建人）
2. 删除类型或部门时，如果已被合规检查使用，系统会阻止删除并提示
3. 类型和部门的排序字段用于控制下拉列表中的显示顺序
4. 启用/禁用状态控制类型和部门是否在前端显示
